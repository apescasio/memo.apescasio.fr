## Context

I wanted to create a development process for my docs/website project `memo.apescasio.fr`. I needed a way to work safely on my local PC while deploying changes quickly to my VPS. The old methods were: manually editing files inside VPS or using FTP to send files inside VPS (time consumer & overall a bad idea). I’d often forget which files I’d changed or accidentally overwrite recent updates.

## Local Git Configuration

The first step was setting up Git on my local PC. Since it wasn’t installed, I used **Winget** to install it. After that, I generated a new SSH key pair using:

```bash
ssh-keygen -t rsa -b 4096
```

I added the contents of public key `mykey.pub` to GitHub’s SSH settings, enabling me to be able to use git clone, git push commands from my local PC.

![Adding SSH Key to GitHub](/images/git/github_ssh_pub.png)

If ever, I want to be able to work on another local PC, I simply need to generate another ssh key and add the public key to Github once again.

## Why is Git important

Git is a safety net for my code. It tracks every change I make, so if something breaks, I can instantly revert to a working version. It’s like having an infinite "undo" button for my entire project. Plus, with GitHub as a central repository, I can work from any computer and always have access to my latest code.

## First `git push`

After setting up Git locally, I created a test file (`test.txt`) and pushed it to GitHub. However, when I checked my VPS, the file wasn’t there. That’s when I realized I needed to configure the VPS side of things.

![First push from local PC](/images/git/github_1st_p.png)

## Setting Up Git Hooks

On my VPS, I created a Git hook called `post-receive` in the `.git/hooks` directory. Initially, it didn’t work as expected. Although the hook was executable (`-rwxr-xr-x`), it wasn’t updating my files. After a few iterations, I simplified the script to fetch the latest changes from GitHub and reset the repository to match the `main` branch exactly.

Here’s the final version of the `post-receive` hook:

```bash title=".git/hooks/post-receive"
#!/bin/bash
# filepath: ~/memo.apescasio.fr.git/hooks/post-receive

### Load nvm and set Node.js version ###
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
nvm use 20.19.0

### Checkout the latest code into the working directory ###
GIT_WORK_TREE=/home/apecio/memo.apescasio.fr git checkout -f main

### Navigate to the working directory ###
cd /home/apecio/memo.apescasio.fr

### Ensure the working directory is clean and up-to-date ###
echo "Ensuring the working directory is clean..."
GIT_DIR=/home/apecio/memo.apescasio.fr/.git GIT_WORK_TREE=/home/apecio/memo.apescasio.fr git fetch origin
GIT_DIR=/home/apecio/memo.apescasio.fr/.git GIT_WORK_TREE=/home/apecio/memo.apescasio.fr git reset --hard origin/main

### Run npm build ###
echo "Running npm build..."
if ! npm install && npm run build; then
    echo "Build failed. Check the logs for details." >> /tmp/deploy.log
    exit 1
fi

### Restart the PM2 process ###
echo "Restarting PM2 process..."
if ! pm2 restart memo; then
    echo "PM2 restart failed ! Check the logs for details." >> /tmp/deploy.log
    exit 1
fi

### Push changes to GitHub ###
echo "Pushing changes to GitHub..."
GIT_SSH_COMMAND="ssh -i /home/apecio/.ssh/id_rsa_memo_github" GIT_DIR=/home/apecio/memo.apescasio.fr/.git GIT_WORK_TREE=/home/apecio/memo.apescasio.fr git push github main

### Log the deployment ###
echo "Deployment completed at $(date)" >> /tmp/deploy.log
```

This ensures my VPS always stays in sync with GitHub. The automation is a game-changer, I can work locally, test everything safely, and deploy with confidence.

## Rollback `git checkout hash`

To revert to the state before adding `test.txt` and `test2.txt`, you would use the following command:

```bash title="git checkout hash"
### This hash specifically as it's the one before the commits of txt files ! ###

git checkout 8b19c6bcff7a1b75aaa7da1bc6210a1b0974dde1
```

## Key Advantages

1. **Version Control**: Git tracks every single changes, allowing me to revert to a previous version if something breaks.
2. **Automation**: My VPS automatically updates whenever I push changes to GitHub from a local PC.
3. **Safety**: I can test changes locally before deploying them to production.

It’s like having a personal assistant that handles all the tedious work of keeping my code in sync, freeing me to focus on writing better code.
