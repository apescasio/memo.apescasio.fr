## ما هو Azure بالضبط؟

**Azure** هو منصة الحوسبة السحابية من مايكروسوفت، ويقدّم خدمات: بنية تحتية كخدمة (IaaS)، منصة كخدمة (PaaS)، وبرمجيات كخدمة (SaaS). يمكّنك من نشر الموارد وإدارتها ومراقبتها داخل مراكز بيانات موزعة عالمياً.

من أكثر الخدمات استخداماً:
- الأجهزة الافتراضية (VM)
- التخزين (Blob, File Share)
- الشبكات الافتراضية (VNet, VPN)
- قواعد البيانات المُدارة (SQL, Cosmos DB)
- الهوية والتحكم بالوصول (Azure AD, RBAC)
- أدوات DevOps والمراقبة والذكاء الاصطناعي ..إلخ

<Callout title="الخلاصة">
Azure سحابة عامة متكاملة يمكنها استبدال أو تكملة البنية التحتية المحلية (On-Premises).
</Callout>

## Azure Active Directory (Azure AD)

**Azure AD** هو خدمة الهوية السحابية من مايكروسوفت. يركّز على:
- مصادقة المستخدم (SSO, MFA …)
- إدارة المجموعات والأدوار والصلاحيات
- التكامل مع آلاف تطبيقات SaaS (مثل: Office 365, Salesforce, GitHub)
- المزامنة مع Active Directory محلي (عبر Azure AD Connect)

هو **ليس** Domain Controller تقليدياً، بل **دليل سحابي** قائم على هويات المستخدمين.
يدعم بروتوكولات قياسية: **OAuth2, OpenID Connect, SAML, WS-Fed**.

<Callout title="حالة استخدام شائعة">
مصادقة مستخدمي Office 365، إدارة الوصول للتطبيقات الداخلية، أو تطبيق MFA عبر السياسات الشرطية (Conditional Access).
</Callout>

## ما هو المستأجر (Tenant) في Azure؟

المستأجر **Azure Tenant** هو نسخة معزولة من Azure Active Directory خاصة بمؤسسة.
يُعد **سياق أمني** يضم:
- المستخدمين
- التطبيقات
- المجموعات
- الأدوار وسياسات الأمان

كل مستأجر يحتوي:
- اسم نطاق افتراضي: `<name>.onmicrosoft.com`
- معرف فريد (GUID) يسمى: `Directory (tenant) ID`
- ويمكن ربطه بعدة اشتراكات Azure (Subscriptions)

<Callout title="مهم">
الاشتراك (Subscription) يوجد داخل مستأجر، لكن يمكن أن يوجد مستأجر بدون أي اشتراك. الاثنان ليسا الشيء نفسه.
</Callout>
