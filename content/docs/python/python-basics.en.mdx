## Python: What Exactly Is It?

**Python** is a high-level, interpreted programming language known for its readability and simplicity. It's widely used for automation, web development, data analysis, and system administration.

Key characteristics:
- Easy-to-read syntax
- Interpreted (no compilation needed)
- Cross-platform (Windows, macOS, Linux)
- Extensive standard library
- Huge ecosystem of third-party packages

<Callout title="Key Takeaway">
Python is perfect for beginners and powerful enough for complex enterprise applications.
</Callout>

## API Basics with Requests

### Making HTTP Requests
```python
import requests
import json

### GET request - Fetch data ###

response = requests.get('https://jsonplaceholder.typicode.com/users')

if response.status_code == 200:
    users = response.json()
    print(f"Found {len(users)} users")
    
    # Print first user's info
    first_user = users[0]
    print(f"Name: {first_user['name']}")
    print(f"Email: {first_user['email']}")
else:
    print(f"Error: {response.status_code}")
```

import requests

### Real-World Example: Server Monitoring
```python
import requests
from datetime import datetime

def check_website_status(url):
    """Check if a website is responding"""
    try:
        response = requests.get(url, timeout=5)
        return {
            'url': url,
            'status_code': response.status_code,
            'response_time': response.elapsed.total_seconds(),
            'is_up': response.status_code == 200,
            'checked_at': datetime.now().isoformat()
        }
    except requests.exceptions.RequestException as e:
        return {
            'url': url,
            'error': str(e),
            'is_up': False,
            'checked_at': datetime.now().isoformat()
        }

### Monitor multiple websites ###

websites = [
    'https://httpbin.org/status/200',
    'https://httpbin.org/status/404',
    'https://httpbin.org/delay/1'
]

print("Website Status Report")
print("-" * 50)

for site in websites:
    status = check_website_status(site)
    
    if status['is_up']:
        print(f"✅ {site} - OK ({status['response_time']:.2f}s)")
    else:
        print(f"❌ {site} - DOWN")
        if 'error' in status:
            print(f"   Error: {status['error']}")
```

## Error Handling

```python
import requests

def safe_api_call(url):
    """Make API call with proper error handling"""
    try:
        response = requests.get(url, timeout=10)
        response.raise_for_status()  # Raises HTTPError for bad responses
        return response.json()
        
    except requests.exceptions.Timeout:
        print("Request timed out")
    except requests.exceptions.ConnectionError:
        print("Connection error")
    except requests.exceptions.HTTPError as e:
        print(f"HTTP error: {e}")
    except requests.exceptions.RequestException as e:
        print(f"Request failed: {e}")
    except json.JSONDecodeError:
        print("Invalid JSON response")
    
    return None

### Usage ###

data = safe_api_call('https://jsonplaceholder.typicode.com/users/1')
if data:
    print(f"User: {data['name']}")
```

<Callout title="Best Practice">
Always include error handling in production scripts. Network requests can fail for many reasons!
</Callout>
